Next Day Eligibility — Installation Guide

1. About this Extension

Next Day Eligibility automates a Yes/No Next Day Eligible attribute on every product based on real-time stock quantity, and restricts configured next-day shipping methods at checkout when any cart item is not eligible.

Products fulfilled by a drop-ship supplier can be exempted from this restriction via a per-product Drop-Ship Eligible flag — even when local stock is zero, those products remain next-day eligible because the supplier ships directly.

Storefront product detail page — the green “Next Day Eligible” badge is rendered just below the price.

Key Features

  • Auto-managed next_day_eligible attribute, recomputed on stock save and product save
  • Per-product drop_ship_eligible override for supplier-shipped goods
  • Green / grey eligibility badge on the product detail page
  • Configurable shipping method codes blocked at checkout when any cart item is ineligible
  • Multi-shipping aware — eligibility evaluated per shipping address
  • Configurable, grouped, and bundle parents propagate from their child items
  • Per-store-view configurable badge labels and method codes

3. Pre-Installation Checklist

Complete these steps before extracting any files. They protect your store from broken caches and partial installs.

1Back up your Magento installation

Take a full backup of app/code, app/etc, and your database. The Adobe Commerce backup tool, mysqldump, or your hosting provider’s snapshot tool all work.

2Set Magento to developer or default mode

Production mode requires extra steps for static content deployment. Switch to developer mode for installation:

php bin/magento deploy:mode:show
php bin/magento deploy:mode:set developer

You can return to production mode after installation completes.

3Disable cache before installing

Disable all caches via the admin: System → Tools → Cache Management, select all, and choose Disable. Or run:

php bin/magento cache:disable
4Verify maintenance access

Make sure you have SSH or terminal access to the server that runs Magento. Manual upload alone is not sufficient — you must run CLI commands to enable the module.

Note: Running setup:upgrade while customers are checking out can cause errors on their carts. Schedule the install during a low-traffic window or enable maintenance mode (php bin/magento maintenance:enable).

4. Installation — Method A: Manual Upload

Use this method if you received the module as a .zip archive (purchased directly from eTechFlow or transferred manually).

1Unpack the archive

Extract etechflow-module-next-day-eligibility-1.0.0.zip on your local machine. Inside you will find the ETechFlow/NextDayEligibility/ folder.

2Create the module directory on your server

SSH to your Magento server. From the Magento root:

mkdir -p app/code/ETechFlow
3Upload the extension files

Upload the NextDayEligibility folder (everything you extracted) into app/code/ETechFlow/. Use SFTP, rsync, or your IDE’s deploy feature.

Final path on the server should be:

app/code/ETechFlow/NextDayEligibility/registration.php
app/code/ETechFlow/NextDayEligibility/etc/module.xml
... (rest of the module files)
4Enable the module

From the Magento root, run:

php bin/magento module:enable ETechFlow_NextDayEligibility
php bin/magento setup:upgrade
php bin/magento setup:di:compile
php bin/magento setup:static-content:deploy -f
php bin/magento cache:flush

For multi-language stores, append every locale code to the deploy command, e.g. setup:static-content:deploy -f en_US en_GB.

Done. Skip ahead to section 6 (Post-Installation Verification).

6. Post-Installation Verification

After installation, confirm the module is loaded and working before continuing to configuration.

6.1 Verify the module is enabled

php bin/magento module:status ETechFlow_NextDayEligibility
# Expected output: Module is enabled

6.2 Verify the data patches ran

Two product attributes should now exist: next_day_eligible and drop_ship_eligible. Check via SQL:

SELECT attribute_code FROM eav_attribute
 WHERE attribute_code IN ('next_day_eligible', 'drop_ship_eligible');

6.3 Visit any product detail page

You should see a green “Next Day Eligible” or grey “Standard Delivery Only” badge directly below the price. If the badge does not appear, hard-refresh the page (the static content may be cached) and verify the cache was flushed.

7.2 Drop-Ship Exception

The Drop-Ship Exception group contains an explanatory note. The actual per-product flag is on the product edit page (see section 8.2).

Finding shipping method codes: Method codes follow the pattern carrier_method (e.g. flatrate_flatrate, ups_NEXTDA). Inspect the quote_shipping_rate table for the exact codes used by your carriers, or read them from Stores → Configuration → Sales → Delivery Methods.

8. Using the Extension

8.1 How eligibility is calculated

Each time a stock item or product is saved, the observer applies this rule:

next_day_eligible = (qty > 0 AND stock_status = in_stock)
                    OR (drop_ship_eligible = Yes)
Stock stateDrop-Ship EligibleResult
In stock with qty ≥ 1—Eligible
Out of stock or qty = 0NoNot eligible
Out of stock or qty = 0YesEligible (supplier ships directly)

8.2 Setting Drop-Ship Eligible per product

  1. Open the product in Catalog → Products.
  2. Scroll to the eTechFlow Shipping attribute group.
  3. Set Drop-Ship Eligible to Yes.
  4. Click Save.

The product save observer immediately re-evaluates eligibility, so customers who already have the product in their cart will see the updated state on their next checkout page reload.

8.3 Configurable, grouped, and bundle products

Parent products inherit eligibility from their children: the parent is eligible if at least one child is eligible, otherwise not eligible. The badge on the parent product page reflects this aggregate.

8.4 Checkout behaviour

If any item in the customer’s cart has next_day_eligible = No, all method codes listed in the configuration are removed from the available shipping rates. This applies to the standard one-page checkout, the REST API, and any headless integration that uses Magento’s shipping rate calculation.

Multi-shipping checkout: When a customer ships items to multiple addresses, each address is evaluated independently. An address with all-eligible items keeps next-day shipping options, even if a different address in the same order has a non-eligible item.

9. Uninstallation

9.1 Remove product attributes (optional)

If you want to delete the eligibility attributes and all their stored data:

php bin/magento eav:attribute:remove next_day_eligible
php bin/magento eav:attribute:remove drop_ship_eligible

9.2 Disable and remove the module

php bin/magento module:disable ETechFlow_NextDayEligibility
composer remove etechflow/module-next-day-eligibility   # Composer install only
php bin/magento setup:upgrade
php bin/magento setup:di:compile
php bin/magento cache:flush

If installed manually, also delete the directory:

rm -rf app/code/ETechFlow/NextDayEligibility

10. Troubleshooting

“Next Day Eligible” badge does not appear

  • Hard-refresh the page (Cmd+Shift+R or Ctrl+Shift+R) — your browser may have cached the old page.
  • Run php bin/magento cache:flush.
  • Verify the module is enabled: php bin/magento module:status ETechFlow_NextDayEligibility.
  • Check var/log/exception.log for module-related errors.

Next-day method is still showing despite OOS items

  • Verify the method code in Stores → Configuration → eTechFlow → Next Day Eligibility matches the actual carrier/method code (e.g. ups_NEXTDA not ups_nextda).
  • Confirm the OOS product is not flagged Drop-Ship Eligible = Yes (which intentionally bypasses the restriction).
  • Reindex: php bin/magento indexer:reindex.

Eligibility didn’t update after I changed Drop-Ship

  • The product save observer recomputes eligibility on every save where the drop-ship value changes. If your save did not actually change the value, no recompute runs.
  • Force a refresh by saving the stock item (any qty change works) — the stock observer recomputes too.
